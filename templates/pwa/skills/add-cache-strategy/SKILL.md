# Add a Cache Strategy

## When to use

When adding offline support or improving load performance by caching assets, API responses, or pages. Activate when the user asks about "caching", "offline support", "service worker", "Workbox", or "faster loads."

## Prerequisites

- Service worker registered in the app (`navigator.serviceWorker.register`)
- `workbox-webpack-plugin` or `vite-plugin-pwa` installed (or plain Workbox CDN)
- HTTPS (or localhost) — service workers require a secure context

## Strategy Decision Guide

| Resource type | Strategy | Why |
|--------------|----------|-----|
| App shell (HTML, CSS, JS bundles) | Cache-first + precache | Never changes between deploys; precache updates on each build |
| Static images, fonts | Cache-first | Rarely changes; stale is fine |
| API responses (real-time data) | Network-first | Freshness required; fall back to cache if offline |
| News / content pages | Stale-while-revalidate | Show instantly, update in background |
| User-uploaded images | Cache-first | Expensive to re-fetch |
| Auth endpoints | Network-only | Never cache credentials or tokens |

**Rule:** Use at most 2-3 strategies per app. Mixing 5+ strategies makes debugging harder than it's worth.

## Steps

### 1. Install Workbox

```bash
# With vite-plugin-pwa (recommended for Vite projects)
npm install -D vite-plugin-pwa

# Or standalone Workbox
npm install workbox-precaching workbox-routing workbox-strategies workbox-expiration
```

### 2. Configure vite-plugin-pwa (Vite projects)

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import { VitePWA } from "vite-plugin-pwa";

export default defineConfig({
  plugins: [
    VitePWA({
      registerType: "autoUpdate",
      workbox: {
        // Precache all assets generated by the build
        globPatterns: ["**/*.{js,css,html,ico,png,svg,woff2}"],

        // Runtime caching rules (applied in order — first match wins)
        runtimeCaching: [
          {
            // API responses — network-first, 24h cache, 50 entries max
            urlPattern: /^https?:\/\/.*\/api\//,
            handler: "NetworkFirst",
            options: {
              cacheName: "api-cache",
              expiration: {
                maxEntries: 50,
                maxAgeSeconds: 24 * 60 * 60,
              },
              networkTimeoutSeconds: 5,
            },
          },
          {
            // Images — cache-first, 30 days, 60 entries max
            urlPattern: /\.(?:png|jpg|jpeg|svg|gif|webp)$/,
            handler: "CacheFirst",
            options: {
              cacheName: "image-cache",
              expiration: {
                maxEntries: 60,
                maxAgeSeconds: 30 * 24 * 60 * 60,
              },
            },
          },
          {
            // Google Fonts stylesheets — stale-while-revalidate
            urlPattern: /^https:\/\/fonts\.googleapis\.com/,
            handler: "StaleWhileRevalidate",
            options: { cacheName: "google-fonts-stylesheets" },
          },
          {
            // Google Fonts files — cache-first, 1 year
            urlPattern: /^https:\/\/fonts\.gstatic\.com/,
            handler: "CacheFirst",
            options: {
              cacheName: "google-fonts-webfonts",
              expiration: {
                maxEntries: 30,
                maxAgeSeconds: 365 * 24 * 60 * 60,
              },
            },
          },
        ],
      },

      manifest: {
        name: "My App",
        short_name: "App",
        theme_color: "#ffffff",
        icons: [
          { src: "pwa-192x192.png", sizes: "192x192", type: "image/png" },
          { src: "pwa-512x512.png", sizes: "512x512", type: "image/png" },
        ],
      },
    }),
  ],
});
```

### 3. Manual Workbox service worker (if not using vite-plugin-pwa)

```javascript
// public/sw.js
import { precacheAndRoute } from "workbox-precaching";
import { registerRoute } from "workbox-routing";
import { NetworkFirst, CacheFirst, StaleWhileRevalidate } from "workbox-strategies";
import { ExpirationPlugin } from "workbox-expiration";

// Precache the app shell (manifest injected by Workbox at build time)
precacheAndRoute(self.__WB_MANIFEST);

// API: network-first
registerRoute(
  ({ url }) => url.pathname.startsWith("/api/"),
  new NetworkFirst({
    cacheName: "api-cache",
    networkTimeoutSeconds: 5,
    plugins: [new ExpirationPlugin({ maxEntries: 50, maxAgeSeconds: 86400 })],
  })
);

// Images: cache-first
registerRoute(
  ({ request }) => request.destination === "image",
  new CacheFirst({
    cacheName: "image-cache",
    plugins: [new ExpirationPlugin({ maxEntries: 60, maxAgeSeconds: 2592000 })],
  })
);

// Navigation (HTML pages): network-first
registerRoute(
  ({ request }) => request.mode === "navigate",
  new NetworkFirst({ cacheName: "pages-cache" })
);
```

### 4. Register the service worker

```typescript
// src/main.ts
import { registerSW } from "virtual:pwa-register";

// vite-plugin-pwa exports this virtual module
const updateSW = registerSW({
  onNeedRefresh() {
    // Show a toast: "New version available — click to update"
    showUpdateBanner(() => updateSW(true));
  },
  onOfflineReady() {
    console.log("App ready to work offline");
  },
});

// Manual registration (without vite-plugin-pwa):
if ("serviceWorker" in navigator) {
  window.addEventListener("load", () => {
    navigator.serviceWorker.register("/sw.js").catch(console.error);
  });
}
```

### 5. Set cache size limits for iOS Safari

iOS Safari enforces a 50MB storage limit per origin and can evict caches without warning.

```javascript
// In your service worker
import { CacheableResponsePlugin } from "workbox-cacheable-response";

// Only cache successful responses (status 200)
new CacheFirst({
  cacheName: "image-cache",
  plugins: [
    new CacheableResponsePlugin({ statuses: [200] }),
    new ExpirationPlugin({
      maxEntries: 30,        // lower for iOS
      maxAgeSeconds: 604800, // 7 days instead of 30
    }),
  ],
})
```

### 6. Test the cache strategy

```bash
# Build with service worker
npm run build

# Serve the built output (service workers need a server, not file://)
npx serve dist

# In Chrome DevTools:
# 1. Application → Service Workers — check "Offline" checkbox
# 2. Reload the page — assets should load from cache
# 3. Application → Cache Storage — verify entries and sizes
# 4. Network tab — "(from ServiceWorker)" label on cached responses
```

### Cache invalidation on deploy

```javascript
// sw.js — clean up old caches when a new service worker activates
self.addEventListener("activate", (event) => {
  const CURRENT_CACHES = ["api-cache", "image-cache", "pages-cache"];
  event.waitUntil(
    caches.keys().then((cacheNames) =>
      Promise.all(
        cacheNames
          .filter((name) => !CURRENT_CACHES.includes(name))
          .map((name) => caches.delete(name))
      )
    )
  );
});
```

## Checklist

- [ ] Service worker registered before page `load` event fires
- [ ] App shell precached — `/` loads without network
- [ ] API responses have a network timeout (5s) so offline fallback fires promptly
- [ ] Image cache has an entry limit (50MB iOS budget, leave headroom)
- [ ] Auth routes use `NetworkOnly` — tokens never cached
- [ ] Old cache versions cleaned up on service worker activation
- [ ] Tested in Chrome DevTools with "Offline" checked
- [ ] "Update available" UI shown when new service worker is waiting

## Files involved

| File | Action |
|------|--------|
| `vite.config.ts` | Add `VitePWA` plugin with `workbox.runtimeCaching` |
| `public/sw.js` | Create (manual) or auto-generated (vite-plugin-pwa) |
| `src/main.ts` | Register service worker or call `registerSW()` |
| `public/manifest.json` | Required for installability (handled by VitePWA or manual) |

## Common mistakes

**Caching POST requests** — Workbox only caches GET requests by default. POST (mutations, login) should always be `NetworkOnly`. Caching a POST response serves the same stale body to every future request.

**No expiration on image cache** — without `ExpirationPlugin`, the image cache grows forever. iOS Safari will evict your entire app's cache (not just the large entries) when storage pressure hits.

**Registering the service worker inside `DOMContentLoaded`** — too late. Register inside `window.load` or at the top level. `DOMContentLoaded` fires before scripts execute; `load` fires after all resources, which avoids interfering with initial page performance.

**Forgetting `networkTimeoutSeconds` on NetworkFirst** — if the network hangs (captive portal, very slow connection), NetworkFirst waits forever before falling back to cache. Always set a timeout of 3–5 seconds.

**Serving from `file://`** — service workers don't work on `file://` URLs. Always test from `localhost` or HTTPS.
